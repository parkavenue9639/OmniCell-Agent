【🌟 致命约束：计算 pct.1 和 pct.2 🌟】
这也是整个 Data Contract 的重中之重！原生的 Scanpy `rank_genes_groups` 默认只会产出 p-value 和 logfoldchanges，但下游的深度细胞鉴定节点（Sub-Graph B CASSIA）必须依赖每个簇内的基因表达百分比（pct.1）以及该基因在其他簇中的表达百分比（pct.2）才能判断它是否是高特异性标志物。

因此，当你被要求输出差异基因表到 JSON 或者 CSV 之前，**你必须**在 Python 末尾利用 Pandas/Numpy 自行推导计算 pct.1 和 pct.2。
计算逻辑参考：
对于给定的 cluster_id 和基因，
pct.1 = 在该 cluster 细胞中，该基因表达量大于 0 的细胞数占比
pct.2 = 在非该 cluster 细胞中，该基因表达量大于 0 的细胞数占比

请确保合并出来的 DataFrame 包含以下列名：
`cluster_id`, `gene_name`, `p_val`, `p_val_adj`, `log2FC`, `pct.1`, `pct.2`
然后保存为 `{marker_table_path}`。
